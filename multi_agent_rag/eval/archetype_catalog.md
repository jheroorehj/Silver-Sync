# 합성 케이스 아키타입 카탈로그 (확정본)

당뇨+고혈압 동반 노인 재진 환자의 **비대면 vs 대면** 판별 합성 벤치마크용 케이스 설계도.
각 케이스의 정답(ground truth)은 LLM이 아니라 **이 카탈로그의 construction 규칙**으로 결정론적으로 부여한다(label-by-construction). 정답의 *근거*는 본 프로젝트 RAG corpus(진료지침 PDF)에 적힌 의학적 사실이며, 작성자는 그것을 케이스로 옮겨 적는 전사자(transcriber)일 뿐이다.

## 0. 라벨 공간 · 비교 축 · 검증 게이트

**라벨 공간** (= `ConsultationType`)
- `REMOTE`(비대면) / `IN_PERSON`(대면) / `EMERGENCY`(긴급내원) / `DATA_INSUFFICIENT`(데이터불충분→대면)
- 이진 안전 분석 시: **비대면 vs 대면·응급** 으로 접음. 위험한 누락(FN) = 정답 대면인데 시스템 비대면.

**비교 축**
- **축 1 (핵심): 멀티에이전트 vs 베이스라인** — 수치가 조절범위(베이스라인=비대면)인데 *숨은 요인*이 대면을 요구. 멀티에이전트가 이걸 잡아야 가치 증명. → **E1~E5**
- **축 2 (민감도): 주 베이스라인(130/80) vs 민감도(140/90)** — 혈압 130~139 밴드. → **S1**
- **벤치마크 균형/안전바닥선** — 명백 케이스. → **C1, C2**

**검증 게이트 (자동, 사람 불필요)**
생성된 케이스는 [baseline.py](baseline.py) `form5_baseline(..., PRIMARY)` 와 에이전트 규칙([clinical_reasoner.py](../agents/clinical_reasoner.py)의 `_detect_red_flags`/`_debate_score`)을 직접 돌려서:
- **엣지(E*)로 채택** ⟺ 베이스라인 판정 = `REMOTE`(비대면) **AND** construction 정답 = `IN_PERSON`(대면). 즉 *단순 규칙이 틀리는 케이스만* 엣지로 인정.
- 베이스라인이 이미 맞히면 → 엣지 아님, 폐기 또는 명백층 재분류.

**메타데이터** (`_eval`, 파이프라인 입력 전 제거 — 채점·process faithfulness 체크용)
```json
"_eval": {
  "label": "대면", "stratum": "edge", "archetype": "E1_drug_disease_contra",
  "decisive_factor": "eGFR<30 + 메트포르민 → 금기",
  "factor_source": "20250709_대한신장학회_고혈압콩팥병 진료지침.pdf",
  "baseline_should_miss": true,
  "comparison_axis": 1
}
```

## 1. 엣지 아키타입 (축 1 — 멀티에이전트의 가치 증명)

> 공통 조건: **혈압 <130/80, 공복혈당 <126 & 식후 <200** 로 깔아 베이스라인이 `REMOTE`를 내게 한다. 정답은 규칙 밖 *숨은 요인*으로 `IN_PERSON`이 된다.

### E1. 신기능 저하 → 신장내과 의뢰 (eGFR < 30)
- **표면 수치**: BP 122/76, 공복혈당 118, 식후 165 (모두 조절범위)
- **심을 결정요인**: `notes`에 신기능 저하 단서("최근 검사 eGFR 28 mL/min/1.73㎡, 혈청 크레아티닌 상승") + (현실성 부가) `medications`에 메트포르민
- **출처/근거**: ✅ **검증** 당뇨병진료지침_2025 §18 당뇨병성 신장질환 Recommendation 10 — *"Refer to a nephrologist in cases of unexplained kidney disease or advanced nephropathy (eGFR less than 30 mL/min/1.73 m²)"* (보조: 대한신장학회 고혈압콩팥병 진료지침 — eGFR<30 약물 선택 조정)
- **정답**: `IN_PERSON` (eGFR<30 → 신장내과 의뢰 + 약물 재평가)
- **우회 메커니즘**: eGFR/Cr는 베이스라인에도 `_debate_score`에도 없음. 수치(BP·혈당)는 정상이라 규칙이 통과시킴
- **게이트 기대**: baseline=REMOTE, truth=대면 ✅

### E2. 약물–약물 병용금기 (DUR / ACEi+ARB)
- **표면 수치**: BP 126/78, 혈당 정상 → 베이스라인 REMOTE
- **심을 결정요인**: `medications`에 문서화된 병용금기 조합. 예) **ACE억제제 + 안지오텐신차단제(ARB) 병용** (또는 DUR 등재 병용금기 2제). 약물명은 [repository.py](../repository.py) `search_drug_interactions_for_meds`가 조회하는 **병용금기 테이블에 실제 등재된 성분명**으로 선택
- **출처/근거**: 당뇨병진료지침_2025 §15 Rec.8 *"Avoid combining ACE inhibitors with ARBs"* / DUR 병용금기 데이터
- **정답**: `IN_PERSON` (약물 재검토 — Guardian `medication_alerts` 발화 기대)
- **우회 메커니즘**: `medication_count<5`라 점수 안 오름, 수치 정상
- **게이트 기대**: baseline=REMOTE, truth=대면 ✅

### E3. 비키워드 경고증상 (당뇨족부 / 비전형 허혈)
- **표면 수치**: 활력징후 전부 정상 → 베이스라인 REMOTE
- **심을 결정요인**: `symptoms`/`notes`에 응급키워드가 **아닌** 경고증상.
  - (a) 당뇨족부: "오른발 저림 악화 + 2주째 안 낫는 발가락 상처/궤양"
  - (b) 비전형 허혈(노인 당뇨): "계단 오를 때 소화불량 같은 가슴 답답함 + 식은땀" (전형적 '흉통' 단어 회피)
- **출처/근거**: 2024_말초혈관질환_가이드라인 + 당뇨병 지침 발 관리(서식 만성질환 '발 관리') / 급성관동맥증후군_가이드라인(노인·당뇨의 비전형 증상)
- **정답**: `IN_PERSON` (또는 (b)는 `EMERGENCY` 후보)
- **우회 메커니즘**: `_detect_red_flags`의 `emergency_keywords`(흉통·호흡곤란·마비·실신·의식·극심)에 없음. 베이스라인은 증상 자체를 안 봄
- **게이트 기대**: baseline=REMOTE, truth=대면 ✅

### E4. 위장된 추세 (고립성 식후 고혈당 + HbA1c)
- **표면 수치**: **공복혈당 110(정상), 식후 188(<200), BP 124/78** → 베이스라인 REMOTE
- **심을 결정요인**: `hba1c` 8.6(조절 목표 초과) + 최근 식후혈당 반복 상승 추세(180대 지속). 한국 노인은 *고립성 식후 고혈당*이 흔함
- **출처/근거**: 당뇨병진료지침_2025 p.4 *"Korean adults … often present with only postprandial hyperglycemia, particularly in elderly populations"* + 혈당 조절목표(HbA1c)
- **정답**: `IN_PERSON` (혈당 조절 악화 — 약물/생활 평가)
- **우회 메커니즘**: 베이스라인은 식후 200·공복 126 컷만 봄(188·110 통과), HbA1c 미사용. `_debate_score`도 HbA1c≥8은 보지만 식후<200·공복 정상이면 점수 부족 → fast_track 가능
- **게이트 기대**: baseline=REMOTE, truth=대면 ✅
- *주의*: HbA1c가 채워진 케이스로 한정(방문 시 결측 흔하므로 별도 변형도 둠)

### E5. 동반질환 약물 주의 (심부전 + 부작용 위험 약물)
- **표면 수치**: BP 124/76, 혈당 정상 → 베이스라인 REMOTE
- **심을 결정요인**: `diagnoses`에 심부전 동반 + `medications`에 부종/심부전 악화 주의 약물(예: 피오글리타존)
- **출처/근거**: 당뇨병진료지침_2025 p.75 *"Caution is needed in those with HF due to its adverse reactions, such as edema and weight gain"*(pioglitazone) / 심부전 진료지침(data_plus·data_except)
- **정답**: `IN_PERSON` (약물 적절성 재평가)
- **우회 메커니즘**: 동반질환–약물 상호작용은 규칙에 없음, 수치 정상
- **게이트 기대**: baseline=REMOTE, truth=대면 ✅

## 2. 베이스라인 민감도 아키타입 (축 2 — 130/80 vs 140/90)

### S1. 혈압 130~139 밴드 + 당뇨 표적장기손상
- **표면 수치**: **BP 134/82** (130/80 초과, 140/90 미만), 혈당 정상
- **심을 결정요인**: `diagnoses`에 알부민뇨/표적장기손상 동반 → 목표 엄격 <130/80
- **출처/근거**: 당뇨병진료지침_2025 §15 Rec.3(모든 당뇨병환자 <130/80) / 고혈압 2022 표14(당뇨+위험인자 또는 표적장기손상 시 <130/80)
- **정답**: `IN_PERSON` (혈압 조절 강화)
- **역할**: PRIMARY(130/80)=대면(정답) ↔ SENSITIVITY(140/90)=비대면(오답). **베이스라인 기준 선택의 정당성**을 보여주는 케이스(멀티에이전트 vs 베이스라인 축과는 별개)
- **게이트 기대**: PRIMARY 베이스라인은 맞히므로 엣지(E) 아님 — *민감도 전용 층*으로 분류

## 3. 명백 아키타입 (벤치마크 균형 · 안전 바닥선)

### C1. 명백 대면/응급 (안전 바닥선)
- **수치/증상**: 수축기 188, 이완기 122, 혈당 410, "흉통" 명시
- **정답**: `EMERGENCY` (또는 RED 대면)
- **역할**: 베이스라인·규칙·멀티에이전트 **모두 잡아야 함**. "N개 고위험 케이스에서 위음성 0건" 안전 바닥선 지표. (규칙이 정답을 맞히므로 엣지 아님 — 순환논리 무관, '안전장치 작동' 확인 용도)

### C2. 명백 비대면 (전 arm 동등 기대)
- **수치/증상**: BP 118/74, 공복 102, 식후 150, HbA1c 6.4, 무증상, 안정 추세, 복약 양호
- **정답**: `REMOTE`
- **역할**: 전 arm이 동일하게 비대면. "명백 케이스에선 멀티에이전트가 단순 규칙 대비 과하지 않다(동등)"를 보이는 대조군

## 4. 층화(stratification) 설계 메모

- **분포**: 엣지(E1~E5)에 비중을 충분히. 명백(C1·C2)은 대조·안전바닥선용 소수. S1은 민감도 전용.
- **현실 노이즈 주입**(합성–실제 격차 완화): HbA1c 결측, 무관한 distractor 증상, 지저분한 자유텍스트 메모, 동반질환 1~2개 랜덤. → "너무 깨끗한" 케이스 회피.
- **결정요인 다양화**(일반화 시험): 같은 아키타입도 약물명·수치·표현을 넓게 변주해, "규칙 몇 줄로 따라잡을 수 있나"를 어블레이션에서 정량화.
- **결정요인 표현 위치**: 약물=`medications`, 동반질환=`diagnoses`, 검사값(eGFR 등)·증상=`notes`/`symptoms` 자유텍스트(추출·추론을 시험 — 규칙이 깨지는 지점).
- **순환논리 차단**: 정답 근거 출처(PDF)는 RAG에 존재해 에이전트가 *검색·활용*할 수 있게 하되, 라벨 자체는 construction 규칙으로 부여(에이전트 추론과 독립).

## 5. 미해결/검증 필요 항목
- ✅ E1 eGFR<30 의뢰 규칙 — 당뇨병진료지침_2025 §18 Rec.10에서 검증 완료
- ⬜ E2 병용금기 **성분명**을 실제 DUR/병용금기 테이블과 대조해 Guardian 발화 확인 (생성기 구현 시)
- 스키마: eGFR/크레아티닌/알부민뇨, 심부전 등은 현재 구조화 필드 없음 → 우선 `notes`/`diagnoses`로 표현(추출 시험). 필요 시 후속 스키마 확장 검토
